# デザインコンセプト - Minimal Shop UI/UX刷新

## 📋 概要

このドキュメントは、Minimal Shopアプリケーションのデザイン刷新における設計思想とデザインコンセプトを記録します。

## 🎯 デザイン目標

### 1. ユーザー体験の向上
- **直感的な操作性**: ユーザーが迷わずに商品情報を閲覧できる
- **視認性の向上**: 重要な情報が一目で理解できる
- **快適な閲覧体験**: 目に優しく、長時間の使用でも疲れにくいデザイン

### 2. モダンでプロフェッショナルな印象
- **クリーンなレイアウト**: 余白を効果的に使用し、情報を整理
- **統一感のあるデザイン**: 一貫したスタイルで信頼性を向上
- **トレンド対応**: 現代的なWebデザインの手法を採用

### 3. アクセシビリティの確保
- **レスポンシブデザイン**: PC、タブレット、スマートフォンに対応
- **高いコントラスト比**: テキストの可読性を確保
- **セマンティックHTML**: スクリーンリーダー対応

## 🎨 デザインシステム

### カラーパレット

#### プライマリーカラー
- **メインカラー**: `#2563eb` (鮮やかなブルー)
  - 用途: ブランドカラー、主要なアクション、リンク
  - 理由: 信頼性と専門性を表現、視認性が高い

- **メインカラー（ホバー）**: `#1d4ed8` (濃いブルー)
  - 用途: ボタンやリンクのホバー状態
  - 理由: インタラクションのフィードバックを明確に

#### セカンダリーカラー
- **アクセントカラー**: `#10b981` (エメラルドグリーン)
  - 用途: 成功メッセージ、ポジティブなアクション
  - 理由: 視覚的な変化を提供し、階層を明確に

#### ニュートラルカラー
- **テキストカラー**: `#1f2937` (ダークグレー)
  - 用途: 本文テキスト
  - 理由: 高いコントラストで可読性を確保

- **セカンダリテキスト**: `#6b7280` (ミディアムグレー)
  - 用途: 補足情報、メタデータ
  - 理由: 視覚的な階層を作る

- **ボーダーカラー**: `#e5e7eb` (ライトグレー)
  - 用途: カード、テーブルの境界線
  - 理由: 柔らかい区切りで圧迫感を軽減

- **背景カラー**: `#f9fafb` (オフホワイト)
  - 用途: ページ背景
  - 理由: 目に優しく、コンテンツを引き立てる

- **カード背景**: `#ffffff` (ホワイト)
  - 用途: カード、モーダル、コンテンツエリア
  - 理由: クリーンで明るい印象

### タイポグラフィ

#### フォントファミリー
```css
font-family: -apple-system, BlinkMacSystemFont, "Segoe UI", Roboto, 
             "Helvetica Neue", Arial, "Noto Sans JP", sans-serif;
```
- **理由**: システムフォントを使用し、各OSで最適な表示を実現
- **日本語対応**: Noto Sans JPをフォールバックに含める

#### フォントサイズスケール
- **見出し1 (h1)**: `2rem` (32px) - ページタイトル
- **見出し2 (h2)**: `1.5rem` (24px) - セクションタイトル
- **見出し3 (h3)**: `1.25rem` (20px) - サブセクション
- **本文**: `1rem` (16px) - 標準テキスト
- **小テキスト**: `0.875rem` (14px) - 補足情報

#### 行間（line-height）
- **見出し**: 1.2 - タイトルの視認性を重視
- **本文**: 1.6 - 読みやすさを確保

### スペーシングシステム

8pxベースのスペーシングシステムを採用：
- **xs**: 0.25rem (4px) - 最小の余白
- **sm**: 0.5rem (8px) - 小さな余白
- **md**: 1rem (16px) - 標準的な余白
- **lg**: 1.5rem (24px) - 大きめの余白
- **xl**: 2rem (32px) - セクション間の余白
- **2xl**: 3rem (48px) - 大きなセクション間の余白

### ボーダーとシャドウ

#### ボーダー半径（border-radius）
- **小**: 0.375rem (6px) - ボタン、入力欄
- **中**: 0.5rem (8px) - カード
- **大**: 0.75rem (12px) - モーダル、大きなコンテナ

#### ボックスシャドウ
- **小**: `0 1px 2px 0 rgba(0, 0, 0, 0.05)` - 控えめな影
- **中**: `0 4px 6px -1px rgba(0, 0, 0, 0.1)` - カードの影
- **大**: `0 10px 15px -3px rgba(0, 0, 0, 0.1)` - モーダルの影
- **ホバー**: `0 10px 25px -5px rgba(0, 0, 0, 0.15)` - インタラクション時

## 📐 レイアウト原則

### 1. カードベースのデザイン
- 商品情報をカードコンポーネントとして表示
- 視覚的なグループ化で情報を整理
- ホバーエフェクトでインタラクティブ性を向上

### 2. レスポンシブグリッドシステム
- **デスクトップ**: 3カラムグリッド
- **タブレット**: 2カラムグリッド
- **モバイル**: 1カラムスタック

### 3. コンテナ幅の管理
- **最大幅**: 1200px（大画面での読みやすさ確保）
- **中央揃え**: コンテンツを中央に配置
- **パディング**: 左右に適切な余白を確保

## 🔧 コンポーネント設計

### ナビゲーションバー
- **位置**: ページ上部に固定
- **背景**: 白背景にシャドウで浮遊感を演出
- **構成**: ロゴ、ナビゲーションリンク、管理画面リンク

### 商品カード
- **レイアウト**: 縦型カード
- **構成**: 商品名、価格、（将来的に画像）
- **インタラクション**: ホバー時に影を強調

### ボタン
- **プライマリボタン**: ブルー背景、白テキスト
- **セカンダリボタン**: 白背景、ボーダー、ブルーテキスト
- **サイズ**: 適切なパディングでタップエリアを確保

### テーブル（管理画面用）
- **スタイル**: ストライプ行で可読性向上
- **レスポンシブ**: 小画面では横スクロール対応

## 🎭 アニメーションとトランジション

### 基本方針
- **スムーズな遷移**: 0.3秒の ease-in-out トランジション
- **控えめな演出**: ユーザーの操作を妨げない程度に
- **パフォーマンス**: CSSトランジションのみ使用

### 適用箇所
- ボタンのホバー状態
- カードのホバーエフェクト
- リンクの色変更

## 📱 レスポンシブデザイン戦略

### ブレークポイント
- **モバイル**: < 640px
- **タブレット**: 640px - 1024px
- **デスクトップ**: > 1024px

### モバイルファースト
- 基本スタイルはモバイルを想定
- メディアクエリで段階的に拡張
- タッチ操作を考慮したUI要素のサイズ

## 🔍 アクセシビリティ配慮

### カラーコントラスト
- WCAG 2.1 AA基準を満たすコントラスト比を確保
- テキストと背景のコントラスト比: 最低4.5:1

### フォーカス管理
- キーボードナビゲーション対応
- フォーカス時の視覚的なインジケーター

### セマンティックHTML
- 適切なHTML5タグの使用（header, nav, main, section, article）
- ARIAラベルの適切な使用

## 📚 参考にしたデザイントレンド

1. **ニューモーフィズム**: 柔らかいシャドウで立体感
2. **ミニマリズム**: 不要な要素を排除し、本質に集中
3. **カードUI**: 情報のグループ化と視覚的な階層
4. **マイクロインタラクション**: 小さなアニメーションでUX向上
